Why package in EasyBuild?

It is a good question, why bother doing this at all?

  • Because we are used to it
  • To perhaps speed deployment of development images

What is RPM?

  • It is a cpio archive with some scripts
  • It is a standard for tracking installed files including tracking
  • It can potentially allow you to relocate software**

How we do it in EasyBuild

  • We leverage a project called FPM (for Effing Package Manager)
  • FPM builds an rpmbuild commandline and .spec file, defining the necessary parts of the spec file
  • We can take advantage of its features to also build for debian

The General Options

Two general options to handle the path that packages will end up and package naming scheme


--package-naming-scheme=PACKAGE-NAMING-SCHEME
        Packaging naming scheme choice (type choice; def
        EasyBuildPNS) (choices: EasyBuildPNS)
--packagepath=PACKAGEPATH
        The destination path for the packages built by
        package-tool (def /home/easybuild/.local/easybuild/packages)

The packaging specific options


        Package options:
          Control packaging performed by EasyBuild. (configfile section package)

          --package           Enabling packaging (def False)
          --package-release=PACKAGE-RELEASE
                              Package release iteration number (def 1)
          --package-tool=PACKAGE-TOOL
                              Packaging tool to use (def fpm)
          --package-type=PACKAGE-TYPE
                              Type of package to generate (def rpm)
